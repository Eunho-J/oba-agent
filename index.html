스스로 연상법을 고치는 개인 에이전트

OBA Agent의 핵심은 더 많은 기억이나 skill을 추가하는 것이 아니라, 같은 기억도 에이전트 자신의 작업 경험에 따라 다르게 떠올리는 recall policy를 스스로 수정하고 검증하는 것입니다. 음성 인식은 whisper.cpp/API STT 어댑터가 맡고, 표현은 EXAONE이, 판단과 실행은 메인 에이전트 API가 맡습니다.

OBA의 진화 대상

skill 목록이 아니라, 기억을 다시 부르는 연상 경로 자체를 candidate로 만들고 검증해 바꿉니다.

Problem

문제는 기억을 저장하는 방식이 아니라, 스스로 떠올리는 방식을 고치지 못한다는 점입니다.

기억을 저장하는 방식과 일하는 방식에는 자유도가 있습니다. 온톨로지, 워크플로우, 벡터 검색, 도구 호출, skill 추가는 계속 좋아지고 있습니다. 하지만 많은 에이전트는 어떤 기억을 어떤 관점으로 먼저 떠올릴지, 그리고 그 방식을 스스로 고치는 능력이 부족합니다.

사람은 같은 단어를 들어도 처한 상황과 경험에 따라 다른 기억을 연상합니다. OBA Agent는 온톨로지/메모리 역할로 설계한 MISO 워크플로우를 호출하고, 에이전트 자신의 작업 경험과 실패 이력에 따라 달라지는 recall policy를 만들고 검증하고 배포하는 것을 핵심 설계 대상으로 둡니다.

Target Users

스스로 달라지는 기억 회상이 필요한 사용자

반복 맥락이 많은 사용자

같은 단어와 요청이 상황에 따라 다른 의미를 갖는 사람. 회의, 구매, 일정, 프로젝트가 대표적입니다.

개인 워크플로우 빌더

새 skill뿐 아니라 어떤 관점으로 회상할지까지 직접 고치고, 에이전트가 스스로 검증하길 원하는 사용자.

한국어 에이전트 사용자

범용 모델의 번역투와 반복 어순에서 벗어나, 작업과 표현이 분리된 자연스러운 대화를 원하는 사람.

Core Features

저장, 회상, 진화, 표현을 분리한 에이전트 구조

01

Contextual Recall Policy

같은 기억도 작업 목적, 상황, 에이전트의 과거 작업 경험에 따라 다른 edge와 관점으로 떠올립니다. OBA의 핵심 자기진화 대상입니다.

02

Self-Evolving Association

에이전트의 작업 경험, 실패, 사용자 피드백을 보고 다음에 어떤 기억을 먼저 떠올릴지 candidate policy를 만들고 검증합니다.

03

Separated Memory

EXAONE이 쓰는 표현/감정 기억과 메인 에이전트가 쓰는 이성/작업 기억을 분리합니다. 서로 연결되지만 권한은 섞지 않습니다.

04

EXAONE Expression Layer

STT 이후의 한국어를 정리하고 범용 모델의 번역투와 기본 말투 회귀를 줄입니다. 생각과 작업은 메인 에이전트가, 한국어 표현과 정서적 전달은 EXAONE이 맡습니다.

05

MISO Workflow Memory

MISO 자체가 메인 두뇌나 memory DB가 아니라, MISO 워크플로우를 온톨로지/메모리 계층처럼 사용합니다. 회상 결과는 메인 에이전트 API가 다시 판단합니다.

Difference

기존 에이전트와의 차이점

연상법 자체가 진화 skill을 추가하는 수준이 아니라, 상황별로 무엇을 먼저 떠올릴지 스스로 후보를 만들고 검증합니다.
일하는 방식 너머 기존 자기개선형 에이전트가 workflow/skill을 진화시킨다면, OBA는 기억을 부르는 경로를 진화시킵니다.
표현과 판단 분리 EXAONE은 사용자와의 소통을, 메인 에이전트는 이성적 판단과 실행을 담당합니다.
두 갈래 메모리 감정/표현 memory와 작업/이성 memory를 분리해 자연스러운 말투와 안전한 실행을 함께 잡습니다.
