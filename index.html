OBA의 목표

기억을 저장하는 방식뿐 아니라, 기억을 다시 부르는 연상 경로 자체를 개인화합니다.

Problem

문제는 기억을 저장하는 방식이 아니라, 기억을 떠올리는 방식입니다.

기억을 저장하는 방식과 일하는 방식에는 자유도가 있습니다. 온톨로지, 워크플로우, 벡터 검색, 도구 호출은 계속 좋아지고 있습니다. 하지만 많은 에이전트는 어떤 기억을 어떤 관점으로 먼저 떠올릴지에는 충분한 자유도가 없습니다.

사람은 같은 단어를 들어도 처한 상황과 경험에 따라 다른 기억을 연상합니다. OBA Agent는 MISO에 저장된 온톨로지를 그대로 검색하는 데서 멈추지 않고, 경험에 따라 달라지는 recall policy를 핵심 설계 대상으로 둡니다.

Target Users

개인화된 기억 회상이 필요한 사용자

반복 맥락이 많은 사용자

같은 단어와 요청이 상황에 따라 다른 의미를 갖는 사람. 회의, 구매, 일정, 프로젝트가 대표적입니다.

개인 워크플로우 빌더

기억 저장뿐 아니라 어떤 관점으로 회상할지까지 직접 고치고 검증하고 싶은 사용자.

한국어 에이전트 사용자

범용 모델의 번역투와 반복 어순에서 벗어나, 작업과 표현이 분리된 자연스러운 대화를 원하는 사람.

Core Features

저장, 회상, 표현을 분리한 에이전트 구조

01

Contextual Recall Policy

같은 기억도 작업 목적, 상황, 사용자 경험에 따라 다른 edge와 관점으로 떠올립니다. OBA의 핵심 개인화 대상입니다.

02

Separated Memory

EXAONE이 쓰는 표현/감정 기억과 메인 에이전트가 쓰는 이성/작업 기억을 분리합니다. 서로 연결되지만 권한은 섞지 않습니다.

03

EXAONE Expression Layer

STT 이후의 한국어를 정리하고 범용 모델의 번역투와 기본 말투 회귀를 줄입니다. 생각과 작업은 메인 에이전트가, 한국어 표현과 정서적 전달은 EXAONE이 맡습니다.

04

MISO Ontology Workflows

MISO는 메인 두뇌가 아니라 경험 memory와 workflow ontology 계층입니다. 회상 결과는 메인 에이전트 API가 다시 판단합니다.

Difference

기존 에이전트와의 차이점

회상 방식 개인화 기억을 저장하는 것에서 멈추지 않고, 상황별로 무엇을 먼저 떠올릴지 조정합니다.
표현과 판단 분리 EXAONE은 사용자와의 소통을, 메인 에이전트는 이성적 판단과 실행을 담당합니다.
두 갈래 메모리 감정/표현 memory와 작업/이성 memory를 분리해 자연스러운 말투와 안전한 실행을 함께 잡습니다.
