응답

좋아요. 앞으로는 후보 비교표를 먼저 띄우고, 확인 전에는 구매를 진행하지 않게 바꿀게요.

Problem

대부분의 에이전트는 말은 잘하지만, 내 작업 환경에 살지 않습니다.

음성 비서는 대화에서 멈추고, 자동화 도구는 사용자의 맥락을 놓칩니다. 개인 에이전트가 유용하려면 내 컴퓨터 자원, 내 워크플로우, 내 말투와 선호를 함께 다룰 수 있어야 합니다.

OBA Agent는 메인 판단 루프를 로컬에 두고, MISO를 지식/워크플로우 계층으로 활용해 "말하는 인터페이스"와 "실제로 일하는 에이전트" 사이의 간극을 줄입니다.

Target Users

복잡한 일을 말로 맡기고 싶은 한국어 사용자

바쁜 개인 사용자

검색, 비교, 예약, 정리 같은 반복 작업을 말로 요청하고 확인하고 싶은 사람.

로컬 툴 사용자

브라우저, 파일, MCP, API를 연결해 자신의 작업 환경에서 바로 실행하고 싶은 사용자.

워크플로우 빌더

에이전트의 처리 방식을 직접 보고, 고치고, 배포 가능한 흐름으로 관리하고 싶은 팀.

Core Features

감성지능과 이성지능을 분리한 가벼운 에이전트 구조

01

EXAONE Expression Layer

한국어 입력을 정리하고 최종 발화를 생성합니다. 감정, 말투, 확인 질문 스타일은 진화 가능한 시스템 프롬프트로 관리합니다.

02

Local Main Agent

판단, 계획, 안전 경계, 툴 실행을 맡습니다. 메인 루프를 로컬에 두어 내 컴퓨터 자원과 MCP를 직접 사용할 수 있습니다.

03

MISO Ontology Workflows

MISO는 메인 두뇌가 아니라 지식 구조와 stateless 워크플로우 계층입니다. 결과는 로컬 에이전트가 다시 판단합니다.

04

ggui Workbench

에이전트가 필요한 UI를 띄우고, 사용자는 워크플로우 노드와 연결, 배포 후보를 눈으로 확인하며 편집할 수 있습니다.

Difference

기존 에이전트와의 차이점

로컬 우선 메인 에이전트가 로컬에 있어 파일, 브라우저, MCP, 앱 제어로 확장할 수 있습니다.
표현과 판단 분리 EXAONE은 사용자와의 소통을, 메인 에이전트는 이성적 판단과 실행을 담당합니다.
자기수정 가능한 워크플로우 프롬프트, 표현층, 온톨로지, 워크플로우를 후보로 만들고 검증 후 반영합니다.
